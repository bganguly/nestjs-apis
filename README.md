# NestJS APIs Labs

A focused collection for NestJS API interview-prep projects.

This folder is intended to hold NestJS-based backend projects so you can practice architecture patterns, compare implementations, and keep notes in one place.

## Core Idea

Build compact NestJS projects around one backend concept at a time and document decisions, tradeoffs, and outcomes.

## Structure

Each project should include:

- Its own README
- Setup and run instructions
- Architectural decisions and tradeoffs
- Optional performance notes

## Suggested Tracks

- Modules, providers, and dependency injection
- Validation, pipes, guards, and interceptors
- JWT auth and role-based access
- CQRS and event-driven patterns
- Testing with unit and e2e coverage
- Production concerns (logging, config, observability)

## How To Use This Index

1. Create or link a NestJS project for one topic.
2. Add setup and run steps in that project.
3. Add the project link and summary in this README.
4. Record what worked and what you would improve.
